# Book Echoes 设计系统规范 v1.0

## 设计理念
本项目视觉语言定位为「档案展陈系统」而非通用 SaaS 产品界面。核心气质：结构化、克制、历史感、终端系统氛围。

---

## 1. 核心色彩体系

| 色彩变量 | 色值 | 使用场景 |
|---------|------|---------|
| `--accent-interactive` | `#C9A063` | **主结构色**：边框、按钮、交互元素、状态指示 |
| `--accent-interactive-light` | `#D4A574` | 高亮文本、标题、强调元素 |
| 深色背景 | `#1a1a1a` | 所有面板、弹窗、容器主背景 |
| 正文文本 | `#D4D4D4` | 常规内容文字 |
| 信息色 | `#7B9DAE` | 代码、引用、辅助信息 |

✅ **设计原则**：金色不是点缀色，是结构骨架色。所有界面的边界、分割、交互入口均以此色构建。

---

## 2. 容器规范

### 2.1 主面板 / 弹窗
```css
/* 标准容器样式 */
background: #1a1a1a / 90%;
backdrop-blur: xl;
border: 1px solid #C9A063 / 20-30%;
```

✅ 规范要点：
- 无圆角，不使用 `rounded-lg/xl/2xl` 类
- 不使用 `box-shadow`，视觉层次完全依靠边框、透明度和留白建立
- 主弹窗必须包含四角装饰元素
- 主面板必须包含顶部标题栏 + 底部状态栏结构

### 2.2 子容器 / 内部面板
```css
background: #1a1a1a / 80%;
border: 1px solid #C9A063 / 15-20%;
```

✅ 规范要点：
- 内层面板透明度降低 10%
- 边框透明度相应降低
- 不需要装饰元素，仅继承色彩体系

## 3. 按钮系统

### 3.1 主按钮 (Primary)
```css
background: #C9A063;
color: #1a1a1a;
border: 1px solid #C9A063;
font-family: font-mono;
letter-spacing: 0.05em;

/* Hover 状态 */
background: transparent;
color: #C9A063;
```

### 3.2 次按钮 (Secondary)
```css
background: transparent;
color: #C9A063;
border: 1px solid #C9A063 / 30%;

/* Hover 状态 */
background: #C9A063;
color: #1a1a1a;
```

### 3.3 辅助按钮 (Tertiary)
```css
background: transparent;
color: #C9A063 / 70% 或 #E8E6DC;
border: 1px solid #C9A063 / 15%;

/* Hover 状态 */
border: 1px solid #C9A063 / 40%;
color: #C9A063;
```

✅ 通用规范：
- 所有操作按钮统一使用 `font-mono tracking-wider`
- 禁止使用 `font-body` 作为按钮字体
- 按钮为直角或极微小圆角，禁止大圆角
- 不使用阴影，状态变化仅通过底色和边框实现

## 4. 排版体系

| 用途 | 字体类 | 备注 |
|------|--------|------|
| 操作按钮、状态标签、系统文本 | `font-mono` | 所有交互控件默认字体 |
| 标题、强调文本 | `font-accent` | 润植家如印奏章楷 |
| 正文内容、说明文字 | `font-info-content` | MiSans 无衬线 |
| 文章、长文本 | `font-body` | 又又意宋 |

✅ 规范要点：
- 按钮、标签、导航、状态栏 **必须** 使用等宽字体
- 仅纯阅读内容使用衬线字体
- 系统感文本添加 `tracking-wider` 增强终端氛围

## 5. 通用组件规范

### 5.1 滚动条
统一使用 `about-overlay-scroll` 类：
- 细滚动条 0.45rem
- 金色系滑块
- 透明轨道
- 禁止为单个页面单独定义滚动条样式

### 5.2 Tooltip / 提示框
```css
background: #1a1a1a;
border: 1px solid #C9A063 / 20%;
font-family: font-info-content;
font-size: xs;
```

✅ 规范要点：
- 无圆角、无阴影
- 设计为「说明面板」而非「气泡提示」
- 箭头颜色与边框体系保持一致

### 5.3 加载 / 运行状态
- 使用金色系弱呼吸、微脉冲、轻扫光动效
- 动效强度克制，无跳动、弹性或高饱和发光
- 动效目的是持续告知运行状态，而非吸引注意力
- 状态变更后立即停止动效

## 6. 主面板标准结构

所有一级弹窗/面板必须遵循此结构：

```
┌─────────────────────────────────┐
│ ■ TITLE [MONO UPPERCASE]     ×  │ ◄── 顶部标题栏
├─────────────────────────────────┤
│                                 │
│  ┌─────────────────────────┐    │
│  │                         │    │
│  │       内容区域          │    │ ◄── 滚动内容区
│  │  (.about-overlay-scroll)│    │
│  │                         │    │
│  └─────────────────────────┘    │
│                                 │
├─────────────────────────────────┤
│ BOOK ECHOES SHANGHAI LIBRARY  ●●●│ ◄── 底部状态栏
└─────────────────────────────────┘
  ◀─ 四角装饰元素 ─▶
```

标准元素清单：
1.  四个角 4x4px 直角边框装饰
2.  顶部标题栏：左侧方块标记 + 等宽大写标题 + 右上角方形关闭按钮
3.  背景水印文字（低透明度 3%）
4.  内容滚动区域使用统一滚动条
5.  底部状态栏：系统标识 + 状态指示点

## 7. 复用检查清单

设计完成后请对照以下标准审核：

✅ **正确**：
- 整体观感像档案系统 / 终端面板
- 金色作为结构边框而非点缀
- 按钮为等宽字体、直边
- 无大圆角卡片
- 没有通用组件库的默认风格

❌ **错误**（需要调整）：
- 使用了 `rounded-lg` 及以上圆角
- 金色仅作为小面积强调色
- 按钮看起来像默认组件库按钮
- 界面有明显阴影层次
- tooltip 是常见的黑色气泡样式
